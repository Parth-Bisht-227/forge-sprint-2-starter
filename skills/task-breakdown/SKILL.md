# Task Breakdown Skill

## Description
Decomposes high-level project goals into granular, actionable technical tasks for the implementation agent.

## Procedure
1. Read `PROJECT.md` and `TASKS.md` to understand the current objective.
2. Analyze the target feature or requirement.
3. Break down the objective into a sequence of atomic steps:
   - **Step 1: Setup/Pre-requisites** (e.g., creating migrations, installing packages).
   - **Step 2: Core Implementation** (e.g., writing the controller, creating the UI component).
   - **Step 3: Validation/Testing** (e.g., running a curl command, checking the browser).
   - **Step 4: Documentation** (e.g., updating `README.md` or `ARCHITECTURE.md`).
4. Ensure each task is "actionable" (starts with a verb and has a clear success criterion).
5. Append the new tasks to `TASKS.md` in the appropriate phase section.
